# CHG-20260510-027 角色服装生成中提示按 Tab 隔离

## 背景

系列详情角色审核页中，重新生成某个角色的默认服装后，切换到同角色其他服装 Tab，例如“新郎”，页面仍显示“正在生成图片...预计需要 30-60 秒”，但同时又能看到该服装已有的正常图片。

## 问题

默认服装生成中且没有新图片 URL 时，模板把 `mainImage` 放在生成中提示容器内部。切换到其他已有图片的服装 Tab 后，前端只更新了图片显示，没有把父级生成中提示容器移除或隐藏，于是出现“正常图片 + 生成中提示”同时存在。

## 改动

1. 将生成中提示改为图片区域内的独立遮罩层。
2. `mainImage` 始终保留在稳定的图片容器中，不再嵌套在生成中提示节点内部。
3. 切换服装 Tab 时，前端可以按当前 Tab 的 `data-generating` 状态正确显示或隐藏遮罩。
4. 当前 Tab 正在生成且没有图片 URL 时，清空主图，避免误显示上一个服装 Tab 的图片。

## 影响范围

- `frontend/templates/series/series_review.html`

## 验证

- Django 模板检查通过。
- 前端页面脚本语法检查通过。
- 前后端已重启。
