# 锁定资产隐藏上传入口

## 编号

CHG-20260510-034

## 背景

已锁定的场景/道具表示当前版本已经定稿，前端仍展示“上传图片”入口会误导用户。后端虽然已经拒绝锁定资产上传新版本，但页面交互也应该保持一致：必须先解锁，才能上传。

## 改动

- 已锁定场景不再显示 hover 区域的“上传图片”按钮。
- 已锁定道具不再显示 hover 区域的“上传图片”按钮。
- 场景/道具三点菜单打开时，如果资产已锁定，会隐藏“上传图片”菜单项。
- 局部锁定/解锁后，不刷新页面也会同步移除或恢复上传入口。
- 菜单点击保留兜底校验，锁定状态下会提示先解锁。

## 后端状态

- 场景上传接口已有 `场景已锁定，无法上传新版本` 校验。
- 道具上传接口已有 `道具已锁定，无法上传新版本` 校验。

## 影响范围

- `frontend/templates/episode/episode_detail.html`
- 后端仅复核，无新增代码。

## 验证

- `python3 manage.py check` 通过。
- `node --check /tmp/episode_detail_script_check.js` 通过。
- `mvn -DskipTests compile` 通过。
- 前后端已重启。
