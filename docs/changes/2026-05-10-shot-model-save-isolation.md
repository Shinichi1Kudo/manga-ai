# CHG-20260510-019 切换视频模型不再覆盖分镜剧情

## 背景

用户在第 2 集 `回门打脸，霸气护妻` 的分镜 7 修改剧情后，切换视频模型时，页面展示又变回包含 `时间【】/镜头【】/剧情【】/音效【】` 的旧格式，表现为“切换模型影响剧情”。

## 根因

分镜列表的统一保存函数同时负责保存剧情、场景和下拉配置。切换模型时也会调用这套保存函数，函数会读取当前剧情编辑器内容，并可能把展示文本解析后一起提交。

另外，分镜资产轮询刷新时只更新了角色/道具绑定和编辑标记，没有同步后端最新的 `description`、`shotType`、`soundEffect` 到编辑器的 `data-*`。当编辑器不在编辑状态时，轮询会用旧数据重新渲染，造成剧情看起来回退。

## 改动

- 新增 `saveShotSettings`，用于保存时长、分辨率、比例、视频模型等下拉配置。
- 视频模型切换只提交 `videoModel`，如模型变化导致分辨率联动，只额外提交 `resolution`。
- 分辨率、比例、时长切换只提交对应设置字段，不再读取或提交剧情内容，也不触发参考图保存。
- 剧情保存时如果用户输入了完整格式文本，只把 `剧情【】` 内的内容落到 `description`，把 `镜头【】` 和 `音效【】` 分别落到独立字段，避免格式壳写入剧情字段。
- 分镜资产轮询刷新时，同步后端最新 `description`、`shotType`、`soundEffect` 到编辑器数据，再重新渲染。

## 数据处理

- 已确认分镜 `707` 后端 `description` 当前为纯剧情：
  `全场死寂，落针可闻。苏雨柔和赵元吓得脸色惨白...`
- 未改动该分镜的视频模型、分辨率、时长等生成配置。

## 验证

- `python3 manage.py check` 通过。
- 前端和后端均已重启，`8000` 与 `8081` 端口正常监听。
