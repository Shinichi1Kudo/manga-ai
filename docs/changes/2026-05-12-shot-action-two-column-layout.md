# CHG-20260512-013 分镜操作区两列布局优化

## 背景

待审核分镜列表左侧区域原本以单列按钮为主，视频预览和操作按钮共用较窄空间。随着预览、重新生成、历史、下载、锁定分镜、上传视频、删除、生成耗时等操作增加，按钮纵向堆叠过长，视觉上拥挤，也影响用户快速扫描。

## 改动

- 将分镜卡片左侧缩略图/操作区加宽到 `w-64`，缩略图改为占满左侧宽度的 `16:9` 预览。
- 将 `.shot-actions` 改为两列网格排列，常用操作按钮按两列排布。
- 对“生成中”“失败”“错误信息”“本次视频生成耗时”等状态提示使用跨两列布局，避免信息被压缩。
- 同步更新 JS 动态渲染的按钮模板，确保生成完成、手动上传、回滚或局部刷新后仍保持同样的两列布局。
- 为操作按钮增加最小高度、图标不收缩和按钮文字不换行规则，减少按钮数量变多时的抖动和拥挤。

## 影响范围

- `frontend/templates/episode/episode_detail.html`
- 影响剧集详情页的待审核分镜列表和由同一分镜卡片移动生成的已锁定分镜列表。
- 不涉及后端接口、数据库结构或视频生成逻辑。

## 验证

- `python3 manage.py check`
- `node --check /tmp/episode_detail_script_check.js`
- `mvn -DskipTests compile`
- 重启前端 `8000` 与后端 `8081`
