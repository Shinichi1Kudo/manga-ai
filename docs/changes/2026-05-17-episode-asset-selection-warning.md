# CHG-20260517-002 剧集解析资产选择弹窗提示与全选联动

## 背景

剧集制作在剧本解析或重新解析后，会弹出场景资产和道具资产选择弹窗。用户需要在这里决定哪些场景、道具要立即生成图片，但原提示不够醒目，容易全选或漏选，导致后续集数用不到的资产也被生成。

同时，弹窗里的“全选场景 / 全选道具”只会向下批量勾选，用户手动取消某个单项后，全选框没有同步成未选中或半选状态，容易让用户误判当前实际勾选范围。

## 改动

- 将资产选择弹窗提示改为带 `必读` 标记的 sticky 高对比提示条，固定在弹窗内容顶部，更靠近用户开始勾选的位置。
- 提示主文案为“请先选择需要的场景和道具”。
- 补充建议“后面集数用不到的道具或场景，不需要勾选生成”。
- 使用 `alert-triangle` 图标、橙红色渐变背景和边框，强化用户注意力。
- 增加场景/道具全选框与单项多选框的双向同步：
  - 点击全选会同步对应组内所有未禁用项。
  - 手动勾选或取消单项后，会实时更新全选框的选中、未选中、半选状态。
  - 没有可选资产时，全选框会自动禁用。
- 调整事件绑定为覆盖式绑定，避免弹窗重复打开后监听器叠加。
- 新增模板测试，约束弹窗必须保留醒目提示锚点、sticky 位置和全选同步脚本。

## 影响范围

- 剧集详情页 `episode_detail.html` 的资产选择弹窗。
- 覆盖首次剧本解析后展示资产选择、重新解析后返回资产选择两个路径。
- 不改变资产生成接口、默认勾选逻辑、扣费逻辑和分镜解析流程。
