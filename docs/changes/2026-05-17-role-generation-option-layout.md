# CHG-20260517-011 添加人物生成方式选项布局优化

## 背景

创建系列和角色审核里添加人物时，人物精细三视图、人物三视图、自行上传人物全身图三个选项原本以三个并列大色块展示，用户容易误以为必须三选一。但实际最推荐的路径是不勾选这些高级选项，直接让系统根据角色提示词自动生成角色图。

## 改动

- 在添加人物表单中新增“推荐使用默认生成”的产品引导。
- 明确默认路径会根据角色描述生成角色三视图，高级方式只在需要指定版式或使用本地人物图时启用。
- 将三个选项收纳到统一的“高级方式”分组中，并给每项增加“可选”标记。
- 高级方式默认折叠，只保留“启用高级方式”开关，用户勾选后才展开三视图/上传图片选项。
- 角色名称字段补充必填星号，让必填状态在视觉上更明确。
- 降低三个选项的视觉权重，避免像必选主流程。
- 创建系列页和系列详情/角色审核页保持一致文案和排版。
- 新增模板测试，约束推荐默认和可选高级方式文案必须保留。

## 影响范围

- 创建系列页 `series_init.html` 的动态角色卡。
- 系列详情/角色审核页 `series_review.html` 的添加新角色弹窗。
- 不改变三视图、上传图片、裁剪、积分和提交接口逻辑。

## 验证

- `python manage.py test apps.series.tests.RoleNameGuidanceTests`
- `python manage.py check`

- 已重启后端 Spring Boot 服务，监听地址 `127.0.0.1:8081`。
