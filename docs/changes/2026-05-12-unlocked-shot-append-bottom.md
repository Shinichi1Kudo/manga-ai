# CHG-20260512-011 解锁分镜持久化到待审核末尾

## 背景

分镜解锁后前端会把卡片移动到待审核分镜列表底部，但后端没有同步更新 `shot_number`。刷新页面或重新进入详情页时，接口仍按旧排序号返回，导致解锁分镜又插回中间。

## 改动

- 后端解锁分镜时查询当前待审核分镜最大排序号。
- 将被解锁分镜的 `shot_number` 写为最大待审核排序号 + 1，确保刷新后仍在待审核列表底部。
- 前端解锁成功后同步使用底部位置的默认分镜编号，用户自定义标题继续保留。

## 影响范围

- 后端：分镜解锁接口、分镜 Mapper。
- 前端：剧集详情页解锁后的本地卡片位置与默认标题显示。
