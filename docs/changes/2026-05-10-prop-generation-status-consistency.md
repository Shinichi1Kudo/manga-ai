# CHG-20260510-007 道具生成状态一致性修复

## 背景

用户反馈：

“在详情页-道具资产点了添加道具，提示词用的铜盆，页面显示生成中，但是打开新页面就看不到这个占位的生成中了；图片已经生成出来了，前端页面下面还显示生成中，应该是待审核。”

排查目标：

- 系列：`重生后，我亲手送夫君下地狱`
- `seriesId=80`
- 剧集：`第1集 血色婚礼变灵堂`
- `episodeId=11`
- 道具：`铜盆`

## 原因

- 新建道具路径只同步写入 `prop` 记录，资产占位 `prop_asset` 在异步生成任务里才创建。用户新开页面时，如果异步任务尚未稳定写入占位资产，就会出现当前页有临时“生成中”卡片、新页面读不到同样占位的情况。
- 异步任务有机会早于创建事务提交开始执行，存在读不到刚创建道具或占位记录的竞态窗口。
- 道具 VO 组装时曾用“最新版本资产 status=0”强行覆盖道具主状态。若存在旧的生成中占位资产，即使已生成出图片并激活，也可能把前端状态压回“生成中”。
- 前端临时道具卡片结构和服务端渲染卡片不一致，轮询拿到完成数据后不一定能找到底部状态文字节点，因此图片已更新但文字仍停留在“生成中”。

## 改动

- 后端 `createProp` 新建道具时同步创建 `prop_asset` 生成中占位记录。
- 后端已存在同名道具重新生成时继续同步创建下一版本占位记录，并复用统一 helper。
- 异步图片生成改为事务提交后再投递到图片线程池，避免生成线程读到未提交数据。
- 新建道具图片生成时复用同步创建的生成中占位资产，成功后把该资产更新为完成、激活，并把道具状态更新为 `待审核`。
- 道具 VO 返回时以 `prop.status` 作为状态源，旧的 `prop_asset.status=0` 不再覆盖已完成的道具主状态。
- 前端添加道具时，等待后端返回 `propId` 后再插入生成中卡片，确保当前页和新页面都依赖同一个后端状态。
- 前端生成中卡片结构改为接近服务端正式卡片，并补充状态文字更新逻辑，轮询完成后能正确显示 `待审核`。
- 前端道具列表为空时仍保留隐藏的 `propsGrid` 容器，添加第一个道具后会移除空状态并显示生成中占位卡。

## 涉及文件

- `backend/src/main/java/com/manga/ai/prop/service/impl/PropServiceImpl.java`
- `frontend/templates/episode/episode_detail.html`

## 行为变化

- 点击添加道具后，后端会立即持久化道具和生成中占位资产。
- 新开同一剧集详情页时，可以从后端列表读到同一个生成中道具占位。
- 图片生成成功后，道具详情和系列道具列表都返回 `status=1`、`statusDesc=待审核`。
- 当前页面轮询到完成状态后会移除遮罩、显示图片，并把底部状态文字改为 `待审核`。
- 原本没有任何道具的剧集，也可以在不刷新页面的情况下展示第一个新增道具的生成中占位。
