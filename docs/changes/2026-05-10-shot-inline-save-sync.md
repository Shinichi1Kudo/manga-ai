# 分镜内联编辑保存同步修复

## 编号

CHG-20260510-011

## 背景

用户在分镜 4 中把音效从 `沉重的心跳声` 改为 `沉重的心跳声1` 后，页面文案里看起来已修改，但后续仍可能使用旧音效；同时保存后部分自动缩略图消失。

## 根因

1. 剧情编辑器保存的是完整文本，例如 `时间【...】 镜头【...】 剧情【...】 音效【...】`，但只写入了 `description` 字段。
2. 后端生成视频提示词时仍读取独立的 `soundEffect` 字段，因此 `description` 里的新音效和 `soundEffect` 里的旧音效可能不一致。
3. 分镜被标记为用户已编辑后，前端只渲染显式 `@{名称}` 资产标记，不再按本分镜已有角色/道具资产做自动缩略图匹配。

## 改动

### 前端

- 保存剧情时解析完整分镜文本中的：
  - `时间【mm:ss-mm:ss】`
  - `镜头【...】`
  - `音效【...】`
- 同步提交 `startTime`、`endTime`、`shotType`、`soundEffect`，避免只更新 `description`。
- 自动渲染出来的缩略图加上 `data-asset-source="auto"`，保存时不会把自动缩略图再写回成 `@{名称}`。
- 手动插入的 `@{名称}` 资产仍按显式引用保存。
- 即使分镜已经被用户编辑，仍会对本分镜已有的角色/道具资产做自动缩略图显示，避免保存后已有缩略图消失。

### 后端

- 生成视频提示词时，如果 `description` 已经是完整结构化分镜文本，会直接使用该完整文本并解析资产引用。
- 完整文本模式下不再重复拼接旧的 `时间`、`镜头`、`剧情`、`音效` 字段，避免新旧字段混用。
