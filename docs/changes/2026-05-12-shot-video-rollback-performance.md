# CHG-20260512-007 分镜视频回滚性能优化

## 背景

待审核分镜列表中，用户从视频历史里回滚版本后，前端会整页刷新。一次回滚成功后又会触发页面详情、用户信息、进度、分镜列表等多次请求；Redis 或接口偶发等待时，用户会感觉点击后十几秒才有反应。

## 改动

- 后端回滚接口从只返回成功状态，改为返回回滚后的 `ShotDetailVO`。
- 前端回滚成功后不再 `location.reload()`，改为局部更新当前分镜卡片：
  - 视频预览
  - 当前视频来源标记
  - 本次视频生成耗时
  - 操作按钮区
- 视频历史弹窗内直接标记新的当前版本，避免用户等待整页刷新。
- 视频历史接口的元数据读取从按版本逐条查询改为批量查询，减少版本较多时的数据库请求数。

## 影响范围

- 后端：分镜视频回滚接口、视频历史版本组装。
- 前端：剧集详情页视频历史回滚交互。
