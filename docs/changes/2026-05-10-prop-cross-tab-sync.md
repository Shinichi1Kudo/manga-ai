# CHG-20260510-014 道具资产跨标签状态同步

## 背景

用户反馈：

“系列《替嫁狂妃：王爷请自重》，第 1 集《重生洞房，一针定情》，在道具资产生成了红烛，但是打开新标签页没看到正在生成中，当前页面有显示，也就是没同步。”

## 日志结论

- `2026-05-10 15:31:04` 前端提交 `POST /api/v1/props/`，后端创建道具 `红烛`，`propId=97`。
- 后端同步创建了占位资产 `assetId=105`，文件名为 `红烛_v1_generating.png`。
- `2026-05-10 15:31:05` 到 `15:31:21` 当前页面轮询 `GET /api/v1/props/97/`，返回仍是生成中。
- `2026-05-10 15:31:25` 图片生成完成，`assetId=105` 更新为完成并激活，道具状态变为 `status=1`、`待审核`。
- 后端当前 `GET /v1/props/series/79` 已能查到 `红烛`、`propId=97`、激活资产 `assetId=105`。

## 原因

后端已经正确落库并保存了生成中占位，但剧集详情页打开新标签时主要依赖服务端首屏渲染和当前页临时卡片。若新标签页打开时首屏上下文没有补齐最新道具，页面不会主动从后端再拉取系列道具状态，因此会错过另一个标签页刚创建的生成中占位或刚完成的待审核状态。

## 改动

- 前端 Django 新增 `GET /api/v1/props/series/<series_id>/` 代理接口，用于详情页按系列补拉最新道具列表。
- 剧集详情页新增道具卡片同步逻辑：
  - 后端返回生成中道具时，自动补齐卡片并启动轮询。
  - 后端返回待审核/已锁定道具时，自动补齐图片、状态文字、历史版本按钮和菜单按钮。
  - 图片完成后把道具写入内联资产映射，并刷新分镜内联缩略图/彩色标签。
- 同步逻辑改为幂等 in-flight，避免多个定时器重复发请求。
- 同步启动点移动到资产映射初始化之后，避免脚本变量初始化顺序导致偶发异常。
- 页面加载后立即同步，1.5 秒和 5 秒各补拉一次；页面可见时每 10 秒同步一次，并在标签页重新可见时立即同步。

## 涉及文件

- `frontend/apps/series/urls.py`
- `frontend/apps/series/views.py`
- `frontend/templates/episode/episode_detail.html`

## 行为变化

- 同一个剧集详情页在不同标签页打开时，道具资产区会从后端补齐最新的生成中/待审核道具。
- 当前标签页生成完成后，新标签页不用依赖刷新时机，也会在短时间内同步到图片和 `待审核` 状态。
- 后续道具图片变化会继续触发分镜列表内联资产刷新，未手动编辑的分镜可拿到最新缩略图和标签。
