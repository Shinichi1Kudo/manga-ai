# CHG-20260510-028 剧集道具上传入口独立拆分

## 背景

剧集制作页的“道具资产”区域里，用户上传本地道具图的入口被放在“添加道具”弹窗里，容易让用户误以为上传也会走火山引擎生成流程。

## 问题

“添加道具”本质是创建道具并调用后端生成图片；“上传道具图片”本质是用户直接提供图片资产，两者动作和预期不同，放在同一个弹窗会造成认知混乱。

## 改动

1. 道具资产标题栏新增独立按钮“上传道具图片”。
2. “添加道具”弹窗移除上传图片区域，只保留 AI 生成所需字段。
3. 新增独立“上传道具图片”弹窗，支持输入道具名、备注、清晰度标记和选择本地图片。
4. 已有道具卡片上的上传图标和右键菜单“上传图片”统一打开上传弹窗。
5. 上传成功后直接更新道具资产卡片，并刷新分镜资产绑定，让分镜可以直接使用上传图。
6. 保留 1:1 校验和裁剪流程；上传路径不调用火山引擎。

## 影响范围

- `frontend/templates/episode/episode_detail.html`

## 验证

- Django 模板检查通过。
- 剧集详情页脚本语法检查通过。
- 前后端已重启。
