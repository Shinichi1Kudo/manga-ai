# CHG-20260517-003 剧本分镜解析提示词增强镜头衔接感

## 背景

剧集制作的分镜解析结果存在“每个分镜各讲各的”的问题。默认版和详细版提示词都强调了拆分、对白保留和视觉细节，但对相邻分镜之间的动作、情绪、视线、声音承接约束不足，导致生成结果容易像独立摘要而不是连续视频。

## 改动

- 为默认版分镜解析提示词增加“连续叙事与镜头衔接”最高优先级规则。
- 为详细版分镜解析提示词增加同一套连续叙事规则。
- 抽出共享提示词片段，避免默认版和详细版以后规则不一致。
- 新规则要求：
  - 先通读整集剧情，建立事件链后再拆分。
  - 第 2 个分镜开始自然承接上一镜的动作、视线、情绪、道具或声音。
  - 每个分镜结尾留下推动下一镜的视觉或情绪钩子。
  - 禁止孤立分镜，相邻分镜必须存在因果、动作延续、视线匹配、反应镜头或声音桥。
  - 连续动作按“动作发起 -> 反应 -> 结果/转折”拆分，对白按“说话者 -> 听者反应 -> 关系变化”拆分。
- 新增后端测试，确保两套提示词都包含连续叙事约束。

## 影响范围

- 剧集制作中“默认版”和“详细版”分镜解析。
- 首次剧本解析后生成分镜、重新解析后生成分镜都会使用新提示词。
- 不影响资产识别、场景/道具生成、视频生成接口和已有分镜编辑保存逻辑。

## 验证

- `mvn test -Dtest=ScriptParseServiceImplTest`
- `mvn -DskipTests compile`

- 已重启后端 Spring Boot 服务，监听端口 `8081`。
