# CHG-20260514-006 首页匿名营销态登录拦截

## 背景

首页现在允许未登录用户访问，用于展示营销内容。但首页上的业务入口按钮会进入需要账号状态的页面，匿名用户点击后体验不明确；同时“我的系列”区域会尝试加载私有系列接口。

## 改动

- 首页业务入口增加登录拦截：
  - 创建新系列
  - 剧集制作
  - 主体替换
  - 我的资产库
  - 积分记录
  - 展示区内的“打开主体替换”
- 未登录用户点击上述入口时不跳转业务页，统一弹出“请先登录”提示，并提供去登录按钮。
- 已登录用户保持原跳转行为不变。
- 未登录状态下不再请求“我的系列”接口，改为展示登录引导卡片。
- 全局登录弹窗函数支持自定义标题、说明和按钮文案，兼容原有“登录已过期”提示。
- 登录中间件放行首页根路径 `/`，确保未登录用户可以看到营销内容。
- “请先登录”提示支持关闭，用户可继续浏览首页营销内容；原“登录已过期”提示保持不可关闭。

## 影响范围

- 前端模板：`frontend/templates/series/series_list.html`
- 基础模板：`frontend/templates/base.html`
- 登录中间件：`frontend/manga_ai/middleware.py`
- 不涉及后端接口和数据库。

## 验证

- `python3 manage.py check` 通过。
- `mvn -DskipTests compile` 通过。
- `git diff --check` 通过。
- 模板关键字检查确认首页业务入口带有登录拦截标记，匿名状态下会显示登录引导卡片。
- 匿名访问 `/` 返回 `200 OK`，可看到首页营销内容，不再跳转到登录页。
- 前端 `8000`、后端 `8081` 已重启并确认监听。
